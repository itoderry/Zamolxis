# Review radar

A weekly pulse on what the public is saying about a named product or brand — across app stores, review sites, forums, and social. The goal is signal: how sentiment splits, what keeps coming up, and anything that needs attention now.

## Know the target first

You need the **product/brand name** and ideally the **sources** to watch (which stores, which sites). Check your memory/profile first. If it isn't set, **ask once and stop** — e.g. *"Which product should I track reviews for, and any specific places — App Store, G2, Reddit, X?"* Save the answer with `memory` (scope "profile") so you don't ask again.

## Build the scan

1. Search for recent mentions with the web search tool(s) — one focused query per source/angle (store reviews, review sites, forum threads, social). Prefer the last week.
2. Keep only real, on-topic results about *this* product. Drop unrelated hits and ads.
3. Read the promising ones rather than trusting the snippet.

## What to report

- **Sentiment split** — roughly how mentions break down (positive / mixed / negative). Be honest it's an estimate from what you found, not a survey.
- **Top themes** — the recurring **praise** and the recurring **complaints**. For each theme, give one **representative quote** and its **source link**.
- **Urgent flags** — call out a complaint spike, a security claim, or anything reputationally hot, up top where it won't be missed.

## Rules

- Only real search results. **Never invent a review, a rating, a quote, or a star count.** Every theme cites a real source link.
- If a source turns up nothing solid this week, say so for that source instead of padding.
- Web search uses DuckDuckGo out of the box; a Tavily or Brave key (Settings → Providers) gives noticeably better coverage here.
